# Security Policy

## 范围

这个仓库包含：

- 可执行脚本
- 可被 Codex 加载进上下文的 skill 文本
- 与多智能体编排、命令执行、artifact 管理相关的模板

因此安全问题不仅包括传统代码漏洞，也包括：

1. 危险脚本或危险命令模板
2. 误导 Codex 执行越权行为的 skill 指令
3. 泄露敏感路径、密钥、令牌的模板或文档
4. 通过 skill 文字诱导“下载并执行”“提权”“删除”之类高风险操作

## 报告方式

如果你发现安全问题，请不要先开公开 issue。

建议至少附上：

1. 受影响文件路径
2. 问题类型
3. 可复现步骤
4. 风险说明
5. 如适用，修复建议

## 当前安全设计原则

本仓库遵循这些原则：

1. 不承诺未验证的原生能力
2. 高风险脚本优先通过 `safe-run.ps1` 审查后执行
3. 默认保留 artifact / audit 证据链
4. 不鼓励隐式、无边界的自动执行
5. 对网络访问、删除、提权、跨工作区写入保持保守

## 不属于漏洞的事项

以下通常不视为安全漏洞，而是产品/平台边界：

1. `implicit` skill discoverability 不稳定
2. Codex 当前会话没有暴露 native child-agent 工具
3. 需要显式 `$skill-name` 才能稳定触发
